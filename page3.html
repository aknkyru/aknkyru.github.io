Sürüm Konrolü

Sürüm kontrolü farklı commit'lerin ve farklı dallar arasında seçim yaparak sağlanır.

Farklı Commit'lere Geçiş

Eğer herhangi bir nedenden dolayı önceki projede önceki sürümlere geçmek gerekirse git log1 komutuyla önceki sürümler görülebilir.

git log komutunun çıktısının bir kısmı

git checkout [hash]2 kullanılarak önceki commit'lere geçiş sağlanabilir. Buradaki hash değeri git log1 komutunun çıktısında her bir commit'ten sonraki commit'i temsil eden veridir. Bu komutu uygulamak içinde bulunduğunuz daldan çıkarır eğer bu commit'ten itibaren değişiklikler yapılmak istenirse yeni bir dal açılarak devam edilebilir.

git reset --[option]3 komutunun kullanımı:

git reset --soft
Bu komut kullanıldığında en son commit'e dönülür ama stage aşamasında veya şu anki dosyalarda bir değişiklik olmaz. Bu sayede unutulmuş küçük değişikler veya commit mesajı yapılandırılarak tekrar commit kaydedilebilir.
git reset --mixed
Bu komut kullanıldığında stage aşamasına eklenenleri kaldırır ama şu anki dosyalarda bir değişiklik olmaz. İstenilen değişiklikler yapılarak tekrar commit yapılabilir.
git reset --hard
Stage aşamasını kaldırır ve şu anki dosyaları bir önceki commit'e çeker. En son yapılan commit'i geri çeker de denilebilir.

Branch(Dal) Kontrolü

Eğer projeye belli bir özellik eklenecekse bu özellik yeni bir dal açılarak bu özelliği ekleyecek kişi bu dal üzerinde ana kodu bozmadan özellik kararlı olana kadar çalışılabilir.git branch [isim]4 komutuna isim yerine dalın ismi girilerek yeni dal oluşturulur.

Projedeki dallar görmek istenilirse git branch4 komutu verilir.

git branch komutunun çıktısı

Dallar arasında geçiş yapılmak istenirse git checkout [dal ismi]2 komutu kullanılır. Örneğin git checkout main2 komutu "main" dalına geçişi sağlar.
