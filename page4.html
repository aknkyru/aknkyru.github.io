Uzak Repository Etkileşimleri

Uzak repository üzerinde birlikte çalışılmak için önceden belirlenmiş başka bir bilgisayar üzerindeki repository'dir.

Uzak Repository'den Güncelleme Almak

Bunu yapmanın 2 yolu vardır:
1. git fetch1
Bu komutu kullanmak uzak repository'deki değişimleri indirir. Ama otomatik olarak birleştirmez. Birleştirmek için git merge [uzak repository]/[dal ismi]2 komutu kullanılır. Uzak repository genelde "origin"dir. Dal ismi şu anki dal ile birleştirmek istediğiniz uzaktaki daldır. İki dalda da aynı kodda bir değişim yapıldıysa bir birleşme anlaşmazlığı(merge conflict) oluşur bu iki değişimden hangisinin saklanmak istediği el ile seçilir daha sonra commit ile kaydedilerek bu anlaşmazlık çözülmüş olur.

Bir diğer yöntem ise git rebase [uzak repository]/[dal ismi]3 komutunu kullanmaktır. Şu anda bulunduğuz dalı belirtmiş olduğunuz başka bir daldaki değişiklikleri sırasıyla bulunduğunuz dala uygular daha sonra bulunmuş olduğunuz dal diğer dalda bulunmayan değişiklikler içeriyorsa bu değişiklikleri uygular.

2. git pull4
Bu komut varsayılan olarak git fetch ve git merge işlemlerini gerçekleştirir.

Uzak Repository'e Güncelleme Yapmak

Bu işlem git push5 komutuyla yapılır. Şu anki seçili dal'ı uzağa yükler. Ancak şu anki dalın uzakta bir karşılığı yoksa öncelikle bu dalın uzaktaki karşılığı olmalıdır eğer yoksa bu komut ile bu dal yüklenebilir: git push [uzak repository] [dal ismi].
git push origin --delete ornek_dal6 bu örnek kullanım kuralları takip edilerek uzaktaki bir dal silinebilir.
